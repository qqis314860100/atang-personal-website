# Browser Tools MCP 一劳永逸使用指南

## 🚀 快速启动

### 方法 1: 使用批处理脚本（推荐）

```bash
# 双击运行
start-mcp.bat
```

### 方法 2: 使用 PowerShell 脚本

```powershell
# 在 PowerShell 中运行
.\start-mcp.ps1
```

### 方法 3: 手动启动

```bash
# 1. 启动服务器
npx @agentdeskai/browser-tools-server@latest

# 2. 新开一个终端，启动客户端
npx @agentdeskai/browser-tools-mcp@latest
```

## 📋 使用步骤

1. **启动 MCP**

   - 运行启动脚本
   - 保持 MCP 窗口打开

2. **安装 Chrome 扩展**

   - 在 Chrome 网上应用店搜索 "Browser Tools"
   - 安装并启用扩展

3. **开始开发**
   - 在 Cursor 中正常开发
   - MCP 会自动检查界面

## 🔧 故障排除

### 问题 1: MCP 工具不可用

**解决方案：**

- 确保 MCP 服务器和客户端都在运行
- 检查端口 3025 是否被占用
- 重启 MCP 服务

### 问题 2: Chrome 扩展未连接

**解决方案：**

- 确保 Chrome 扩展已安装
- 检查扩展是否连接到 localhost:3025
- 刷新浏览器页面

### 问题 3: 截图失败

**解决方案：**

- 确保在正确的浏览器标签页
- 检查页面是否完全加载
- 重启 MCP 客户端

## 💡 最佳实践

1. **保持 MCP 运行**

   - 开发期间不要关闭 MCP 窗口
   - 如需重启，先关闭所有 MCP 进程

2. **定期检查连接**

   - 如果工具不可用，运行启动脚本
   - 检查网络连接状态

3. **使用自动检查**
   - 在 Cursor 中启用自动检查模式
   - 重要修改后自动验证效果

## 🎯 优势

- ✅ **一劳永逸**：一次启动，持续工作
- ✅ **自动检查**：无需手动提醒
- ✅ **实时反馈**：立即看到修改效果
- ✅ **提高效率**：减少重复操作

## 📞 技术支持

如果遇到问题：

1. 检查 MCP 进程是否运行
2. 重启 MCP 服务
3. 检查 Chrome 扩展状态
4. 查看控制台错误信息
